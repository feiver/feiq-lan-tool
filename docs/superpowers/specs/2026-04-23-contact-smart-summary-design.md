# 联系人单行智能摘要设计

## 背景

当前联系人列表已经支持未读数字提示和待接收投递提示，但卡片正文仍偏向设备信息展示，例如主机名、状态和 IP。用户在联系人列表里更需要快速判断“这个联系人现在有什么需要处理”，因此下一步改为联系人级单行智能摘要。

## 目标

- 在联系人卡片中新增一行智能摘要，帮助用户快速定位下一步动作。
- 保留现有未读角标和待接收标签，不破坏已验证的强提示能力。
- 优先展示待处理状态；没有状态时展示最近一条消息预览。
- 不新增后端协议，不改变消息和传输模型，优先从现有前端状态推导。

## 非目标

- 不做多行摘要展开。
- 不在联系人列表展示长文件名、长目录路径或完整 IP 诊断信息。
- 不新增联系人详情页。
- 不改变点击联系人、清除未读、接收/取消文件投递等现有行为。

## 摘要优先级

联系人摘要只显示一行，按以下优先级抢占：

1. 待接收文件
2. 传输中
3. 有未读消息
4. 最近一条消息预览

### 待接收文件

从现有 `messages` 中的 delivery 消息推导。按 `request_id` 聚合投递请求并取最新状态；如果最新状态为 `PendingDecision`，且来源联系人不是本机，则该联系人摘要显示 `待接收文件`。

现有标题区的 `待接收` 标签继续保留。标签负责显眼提醒，摘要负责在卡片正文里说明当前待处理事项。

### 传输中

从现有 `transfers` 推导。只要该联系人与本机之间存在状态为 `InProgress` 的传输任务，且没有更高优先级摘要，则显示 `传输中`。

联系人维度不区分收发方向，只要任务与该联系人相关且正在传输，就认为该联系人处于传输中。

### 有未读消息

复用现有 `contactUnreadCounts`。如果该联系人未读数大于 0，且没有待接收文件或传输中任务，则摘要显示 `有未读消息`。

未读角标继续显示具体数字；摘要只做文本提示，不替代角标。

### 最近一条消息预览

当联系人没有强状态时，从 `messages` 中筛选与该联系人相关的 `direct` 和 `delivery` 消息，取最新一条生成预览。

- 对方发来的文本消息：`对方：<消息内容>`
- 本机发出的文本消息：`你：<消息内容>`
- 对方发来的空文本消息：`对方：发来一条消息`
- 本机发出的空文本消息：`你：发出一条消息`
- delivery 文件消息：`对方：发送了文件` 或 `你：发送了文件`
- delivery 目录消息：`对方：发送了目录` 或 `你：发送了目录`

文件和目录消息不展示长路径或完整文件列表，避免联系人列表噪音过大。

## 界面布局

联系人卡片调整为轻量的三段结构：

- 标题区：昵称、待接收标签、未读角标。
- 摘要区：新增单行智能摘要，放在昵称下方，作为主要说明。
- 辅助信息区：只保留一项低优先级识别信息，优先显示 `status_message`，没有则显示 `host_name`。

`ip_addr` 不再作为联系人卡片主路径展示信息。IP 仍可通过网络诊断、设置或后续设备详情能力查看。

摘要文本视觉上弱于昵称、强于辅助信息；超长内容使用单行省略。

## 交互

- 点击联系人进入会话的行为保持不变。
- 点击联系人后，未读数字清零逻辑保持不变。
- 点击联系人后，如果只有未读状态，摘要应从 `有未读消息` 回落到最近消息预览。
- 点击联系人不会清除待接收摘要；只有接收或取消对应投递后，待接收摘要才消失。
- 传输任务状态更新后，联系人摘要随 `transfers` 重新推导。
- 空联系人列表行为保持不变。

## 数据边界

本次实现不新增持久化字段。建议在 `App.tsx` 里从 `messages`、`transfers`、`contactUnreadCounts` 和 `settings.runtime.deviceId` 推导 `contactSummaries`，再传入 `ContactsPanel`。

联系人面板只负责展示摘要，不负责理解消息或传输状态规则。

## 验收标准

- 联系人存在入站待接收投递时，显示 `待接收` 标签，并显示 `待接收文件` 摘要。
- 联系人存在进行中的传输任务，且没有更高优先级状态时，摘要显示 `传输中`。
- 联系人存在未读消息，且没有待接收或传输中状态时，摘要显示 `有未读消息`。
- 联系人没有强状态时，摘要显示最近一条消息预览。
- 最近消息预览区分方向：对方消息使用 `对方：`，本机消息使用 `你：`。
- 文件和目录消息预览使用短文案，不展示长路径。
- 点击联系人后，未读角标清零，摘要从 `有未读消息` 回落到最近消息预览。
- 点击联系人后，待接收摘要不消失。
- 空联系人列表行为不变。
- 现有会话、文件接收、未读提示测试不回退。

## 测试计划

- 在 `app-shell` 测试中新增联系人摘要用例。
- 覆盖优先级：待接收优先于传输中，传输中优先于未读，未读优先于最近消息，无状态时显示最近消息。
- 覆盖点击联系人后的未读摘要刷新。
- 覆盖 direct 文本消息的方向前缀。
- 覆盖 delivery 文件和目录消息的短文案。
