# 应用内横幅提醒设置开关设计

## 背景

当前应用已经支持单聊消息和文件投递的应用内横幅提醒：当事件来源不是当前会话时，主窗口右上角会显示一条横幅，并提供 `查看` 与 `关闭`。这个能力解决了“用户没有打开对应会话时缺少即时提示”的问题，但还没有进入“聊天与通知”设置页，用户无法关闭这类提醒。

下一步应先把横幅提醒纳入设置系统，形成一个最小可控闭环。更复杂的免打扰、停留时长、声音、按联系人策略暂不进入本轮，避免设置页和状态组合过早膨胀。

## 目标

- 在“聊天与通知”设置页新增 `应用内横幅提醒` 开关。
- 默认开启，保持现有横幅行为不变。
- 当开关关闭时，非当前会话的单聊消息和文件投递不再弹出横幅。
- 开关只影响横幅提醒，不影响消息记录、未读角标、待接收标签、文件投递卡片和自动切换设置。
- 保持旧设置数据兼容：历史配置中没有该字段时按默认开启处理。

## 非目标

- 不新增系统通知。
- 不新增声音提醒。
- 不新增免打扰时段。
- 不支持自定义横幅停留时间。
- 不支持按联系人、群组或消息类型分别配置提醒。
- 不改变广播消息不触发横幅的现有规则。

## 用户体验

### 设置入口

位置：设置中心 -> 聊天与通知。

新增一个开关项：

- 标题：`应用内横幅提醒`
- 说明：`收到非当前会话的单聊消息或文件投递时，在主窗口显示横幅。`

默认值为开启。

### 开启状态

保持当前行为：

- 非当前会话的单聊消息显示 `<联系人名> 发来新消息`。
- 非当前会话的文件投递显示 `<联系人名> 发来文件投递`。
- 当前会话消息不显示横幅。
- 广播消息不显示横幅。
- 新横幅覆盖旧横幅。
- 横幅 6 秒后自动消失。

### 关闭状态

关闭后：

- 单聊消息仍进入会话记录。
- 文件投递仍显示在对应会话中。
- 联系人未读角标仍按现有规则增加。
- 待接收标签仍按现有规则显示。
- 不显示应用内横幅。

## 数据模型

在 `ChatPreferences` 中新增字段：

```ts
showInAppBannerNotifications: boolean;
```

默认值：

```ts
showInAppBannerNotifications: true;
```

兼容要求：

- `preferences.chat` 为空时，继续使用 `defaultChatPreferences`。
- `preferences.chat` 存在但缺少 `showInAppBannerNotifications` 时，应按 `true` 补齐。
- 前端读取设置时不得因为旧配置缺字段导致横幅开关显示为关闭或运行时报错。

## 行为规则

现有 `chat-message-received` 流程保持顺序：

1. 记录消息。
2. 判断自动切换设置；如果自动切换生效，直接切到来源会话并返回，不显示横幅。
3. 判断是否需要增加未读。
4. 判断是否允许显示横幅：
   - 来源是已知设备；
   - 来源不是当前会话；
   - 消息类型是 `direct` 或 `delivery`；
   - `showInAppBannerNotifications` 为 `true`。
5. 满足条件时显示横幅，否则只保留消息和联系人状态更新。

## 测试计划

在 `app-shell` 测试中补充以下用例：

- 设置页显示 `应用内横幅提醒` 开关，默认开启。
- 点击该开关后，`settings.preferences.chat.showInAppBannerNotifications` 更新为 `false`。
- 当开关为关闭时，非当前会话的单聊消息不显示横幅，但联系人未读角标仍增加。
- 当开关为关闭时，非当前会话的文件投递不显示横幅，但联系人待接收提示仍显示。
- 旧配置缺少该字段时，横幅提醒默认开启。
- 保持现有横幅测试通过，确认开关开启时行为不回退。

## 验收标准

- “聊天与通知”设置页可以看到并切换 `应用内横幅提醒`。
- 默认配置和旧配置都保持横幅提醒开启。
- 关闭开关后，单聊消息和文件投递不会再显示应用内横幅。
- 关闭开关不影响未读角标、待接收标签、消息记录和文件投递卡片。
- 自动切换设置仍优先于横幅提醒。
- README 同步记录应用内横幅提醒设置能力。
- `npm.cmd run test -- app-shell`、`npm.cmd run test` 和 `cargo check` 通过后，才可关闭实现任务。
