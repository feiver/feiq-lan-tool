# 设置系统设计

> 实施状态（2026-04-16）：
> - Phase 1 已覆盖：五组设置骨架、结构化设置模型、个人与身份、文件传输、网络与发现
> - 待后续计划覆盖：聊天与通知、显示与通用、传输高级参数
> - 当前本地验证情况：`npm run test -- app-shell` 通过；Rust 侧 `cargo check --tests` 通过；`cargo test` 仍受本机 GNU 链接器影响

## 背景

当前项目的桌面端设置能力仍停留在 MVP 早期阶段，只覆盖了两个本地项：

- 昵称
- 下载目录

但产品目标是参照 FeiQ 的局域网办公心智，逐步形成一套完整、跨平台、可扩展的设置系统。用户已经明确希望设置体系覆盖以下传统心智范围：

- 个人设置
- 系统设置
- 发送接收设置
- 群设置
- 功能设置
- 网络设置
- 显示设置
- 个性设置
- 网段设置

本次设计不直接复刻 FeiQ 的旧式分类，而是采用现代产品域重组方案：保留 FeiQ 的能力覆盖面，但将设置入口收敛为更适合当前项目阶段和普通办公用户的结构。

## 目标

- 建立适合当前项目阶段的设置系统信息架构
- 明确哪些设置属于 V1 必做、V2 增强、V3 扩展
- 将高频设置前置，将复杂参数收敛到“高级设置”
- 保持跨平台实现可行性，不提前引入大量仅 Windows 有意义的遗留开关
- 为后续群组、广播、网络诊断、个性化主题等能力预留设置扩展口

## 非目标

- 本次不实现群设置本身
- 本次不实现插件、自定义消息、垃圾消息屏蔽等增强体系
- 本次不实现 FeiQ 式细粒度热键矩阵和窗口行为矩阵
- 本次不实现管理员控制台级别的系统诊断面板
- 本次不要求一次性将全部设置项落地为可用 UI

## 核心决策

### 1. 采用产品域重组，而不是沿用 FeiQ 原分类

首版设置入口统一重组为 5 个分组：

1. 个人与身份
2. 聊天与通知
3. 文件传输
4. 网络与发现
5. 显示与通用

这样可以降低普通办公用户的理解成本，也更符合当前产品的真实能力边界。

### 2. 默认简洁，高级折叠

设置页默认只展示：

- 高频
- 易理解
- 改完马上能看到结果

的设置项。

复杂网络参数、传输性能参数、兼容性参数全部统一进入“高级设置”折叠区。

### 3. 普通办公用户优先

首版设置优先服务普通办公用户，不以高级网络管理员为主目标人群。因此：

- 文件接收默认手动确认
- 端口默认自动分配，不向普通用户开放修改
- 网络发现保留一级入口，但界面尽量简洁
- 系统设置、个性设置不单独拆组

### 4. 群设置在群功能正式落地前不出现在 UI 中

虽然群能力已经进入中长期规划，但首版不展示“空设置入口”，避免形成无效设置或误导用户。

### 5. 系统设置与个性设置先并入“显示与通用”

开机启动、托盘、窗口关闭行为、主题、字号等行为先统一收敛在“显示与通用”里。首版只开放托盘与关闭行为，主题、字号与排序类显示项延后到后续版本。待功能密度提升后，再在 V3 之后考虑独立拆出“系统设置”与“个性设置”。

## 信息架构

### 侧边栏分组

1. 个人与身份
2. 聊天与通知
3. 文件传输
4. 网络与发现
5. 显示与通用

### 每组统一布局

每个分组统一采用三段结构：

1. 常用设置
2. 高级设置
3. 当前状态

其中：

- 常用设置：直接可改
- 高级设置：默认折叠
- 当前状态：只读展示，用于排障和确认运行状态

## 分组设计

### 1. 个人与身份

#### 目标

定义用户在局域网中的展示身份。

#### V1 字段

- `nickname`
  - 类型：文本输入
  - 默认值：当前电脑名
  - 生效方式：即时生效
- `deviceNameMode`
  - 类型：单选
  - 值：`显示昵称` / `显示昵称（附设备名）`
  - 默认值：`显示昵称`
  - 生效方式：即时生效
- `statusMessage`
  - 类型：文本输入
  - 默认值：空
  - 生效方式：即时生效
- `deviceId`
  - 类型：只读文本
  - 默认值：运行期设备 ID
  - 生效方式：只读展示

#### V2 预留

- 头像
- 签名档
- 部门 / 职位

#### 暂不做

- 多身份切换
- 企业名片详情页

### 2. 聊天与通知

#### 目标

管理消息提醒和会话提示行为。

#### V1 字段

- `messageSoundEnabled`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效
- `messagePopupEnabled`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效
- `broadcastNotifyEnabled`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效
- `historyPreviewCount`
  - 类型：数字输入或下拉
  - 默认值：`6`
  - 生效方式：即时生效
- `unreadIndicatorMode`
  - 类型：单选
  - 值：`角标` / `高亮` / `角标+高亮`
  - 默认值：`角标+高亮`
  - 生效方式：即时生效
- `allowAttentionSignal`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效

#### V2 预留

- 忙碌 / 免打扰
- 指定联系人例外提醒
- 快捷回复 / 自动回复

#### 暂不做

- FeiQ 式逐条弹窗策略矩阵
- 大量快捷键与提示框粒度控制

### 3. 文件传输

#### 目标

管理文件发送、接收、保存和传输参数。

#### V1 字段

- `downloadDir`
  - 类型：目录选择
  - 默认值：系统下载目录
  - 生效方式：即时生效
- `receiveBeforeAccept`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效
- `resumeEnabled`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效
- `openFolderAfterReceive`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效
- `preserveDirectoryStructure`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效

#### V1 高级字段

- `transferChunkSize`
  - 类型：数字输入
  - 默认值：协议安全默认值
  - 生效方式：仅对新任务生效
- `transferRetryCount`
  - 类型：数字输入
  - 默认值：`10`
  - 生效方式：仅对新任务生效
- `transferSpeedLimit`
  - 类型：数字输入
  - 默认值：`0`（不限速）
  - 生效方式：仅对新任务生效

#### V2 预留

- 自动接收白名单
- 小文件自动接收
- 指定联系人自动接收
- 文件完整性校验提示

#### 暂不做

- 默认自动接收全部文件
- 过多底层网络兼容开关

### 4. 网络与发现

#### 目标

控制设备发现方式、网段范围和基础网络参数。

#### V1 字段

- `discoveryMode`
  - 类型：单选
  - 值：`自动发现` / `手动网段辅助发现` / `仅当前网段`
  - 默认值：`自动发现`
  - 生效方式：保存后重载发现服务
- `manualSegments`
  - 类型：列表配置
  - 默认值：空
  - 生效方式：保存后重载发现服务
- `messagePortReadonly`
  - 类型：只读展示
  - 默认值：当前消息端口
- `filePortReadonly`
  - 类型：只读展示
  - 默认值：当前文件端口

#### V1 高级字段

- `multicastAddress`
  - 类型：文本输入
  - 默认值：协议默认值
  - 生效方式：保存后重载发现服务
- `packetSize`
  - 类型：数字输入
  - 默认值：协议默认值
  - 生效方式：对新发现流程生效
- `discoveryRetryCount`
  - 类型：数字输入
  - 默认值：协议默认值
  - 生效方式：对新发现流程生效

#### V2 预留

- 多网卡优先级
- 指定网卡
- 跨子网发现策略
- 网络诊断信息

#### 暂不做

- MAC 地址伪装
- 复杂 Windows 防火墙向导项
- 老式 IP / MAC 混合兼容开关

### 5. 显示与通用

#### 目标

统一承接基础客户端行为，并为后续显示个性化预留扩展口。

#### V1 字段

- `trayEnabled`
  - 类型：开关
  - 默认值：开
  - 生效方式：即时生效
- `closeAction`
  - 类型：单选
  - 值：`最小化到托盘` / `直接退出`
  - 默认值：`最小化到托盘`
  - 生效方式：即时生效

#### V2 预留

- 联系人排序
- 会话排序
- 时间显示格式
- 主题模式
- 字号缩放
- 紧凑 / 舒展布局
- 列表密度
- 更多主题细节

#### 暂不做

- 图标列模板矩阵
- 自定义论坛链接
- 复杂皮肤系统

## 首屏 / 高级 / 只读分层

### 个人与身份

- 首屏：昵称、个人状态、设备名展示方式
- 只读：设备 ID

### 聊天与通知

- 首屏：声音提醒、弹窗提醒、广播提醒、历史消息显示条数
- 高级：未读提示方式、提醒信号开关

### 文件传输

- 首屏：下载目录、接收前确认、断点续传、完成后打开目录、保留目录结构
- 高级：分片大小、重试次数、发送限速
- 只读：默认传输说明

### 网络与发现

- 首屏：发现模式、手动网段管理入口、当前发现状态
- 高级：组播地址、广播包大小、重试次数
- 只读：当前消息端口、当前文件端口

### 显示与通用

- 首屏：托盘显示、关闭行为
- 高级：首版暂不开放
- 只读：首版暂不展示，后续可加入版本号、数据目录等运行信息

## 设置文案

### 侧边栏文案

1. 个人与身份
2. 聊天与通知
3. 文件传输
4. 网络与发现
5. 显示与通用

### 分组副标题

- 个人与身份：设置你在局域网中的展示信息
- 聊天与通知：管理消息提醒、弹窗和未读提示方式
- 文件传输：设置文件接收、保存和传输行为
- 网络与发现：管理设备发现方式和局域网扫描范围
- 显示与通用：调整托盘显示、关闭行为与后续显示体验

### 通用交互文案

- 折叠区标题统一为：`高级设置`
- 只读区标题统一为：`当前状态`
- 操作按钮统一为：`恢复默认`、`保存设置`
- 若需要重载网络服务，提示为：`保存后将重新加载网络发现服务`

## 存储设计

### 当前状态

当前 Rust 侧只存在 `RuntimeSettings`，字段包括：

- `device_id`
- `nickname`
- `download_dir`

### 设计建议

后续设置系统应从单一 `RuntimeSettings` 演进为结构化设置模型，例如：

```text
AppPreferences
- identity
- chat
- transfer
- network
- display
```

每一组设置独立分层，避免未来将大量可选项继续堆到单个顶层结构中。

### 持久化原则

- 所有设置统一持久化到本地配置文件
- UI 修改后先更新前端状态，再同步到桌面侧
- 即时生效项立刻影响运行时
- 网络项在保存后触发网络发现服务重载
- 只读项不参与持久化写入

## 生效规则

### 即时生效

- 昵称
- 个人状态
- 消息提醒
- 下载目录
- 显示排序
- 主题与字号
- 托盘和关闭行为

### 对新任务生效

- 传输分片大小
- 传输重试次数
- 传输限速
- 发现重试次数

### 保存后重载服务

- 发现模式
- 手动网段
- 组播地址

### 只读展示

- 设备 ID
- 当前消息端口
- 当前文件端口

## 验收标准

- 用户能在 5 组结构下快速理解设置范围
- V1 设置项全部有明确用途，不出现空开关
- 高级参数默认收起，不影响普通用户使用
- 设置项具备明确生效规则：即时、对新任务生效、保存后重载、只读
- 当前已实现的昵称与下载目录能够自然归入新结构
- 文案与交互保持跨平台桌面工具风格，不依赖 FeiQ 的历史遗留表达
